# s2-chips-to-map — pinned dependencies
#
# Target environment: Google Colab (Ubuntu 22.04, Python 3.11/3.12, NVIDIA T4).
#
# PINNING POLICY — read this before "tightening" anything here.
#
# Two groups, deliberately treated differently:
#
#   (a) The ABI-coupled scientific core: numpy, scipy, pandas, scikit-learn,
#       matplotlib, torch, torchvision. These ship compiled extensions that are
#       built against a specific numpy C ABI. Colab preinstalls a mutually
#       consistent set of them. Pinning an exact version here does NOT give
#       reproducibility on Colab — it *downgrades a package on disk while the
#       running kernel still holds the old one in memory*, and the next import
#       of any compiled extension dies with
#
#           ValueError: numpy.dtype size changed, may indicate binary
#           incompatibility. Expected 96 from C header, got 88 from PyObject
#
#       (96 = the numpy 2.x dtype struct, 88 = numpy 1.x). So this group gets
#       compatible RANGES, and pip's default "only if needed" resolution leaves
#       Colab's working set untouched.
#
#   (b) Everything else — pure-Python or independently versioned libraries.
#       These are pinned exactly, which is where pinning actually buys
#       reproducibility.
#
# For a byte-identical local environment, install into a fresh venv where the
# ranges below resolve deterministically against a lock file of your choosing.
# ---------------------------------------------------------------------------
